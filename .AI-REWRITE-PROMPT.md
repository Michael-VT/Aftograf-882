# Autograf-882 Debug Simulator — Multi-Language Rewrite Brief

## Project DNA

Browser-based debugger + simulator for the **Autograf-882** (1988 Soviet flatbed plotter).  
CPU: **K580IK80A** (Intel 8080 clone).  
Original team built it in one year — this project preserves, simulates, and documents that work.

## Target Languages (in order)

1. **Rust** — primary, performance-critical core
2. **Python** — bindings/scripting layer
3. **Go** — future cross-platform CLI/server

## Hardware Spec (must implement)

### CPU — К580ИК80А (Intel 8080)
- 256 opcodes, table-driven execution
- Registers: A, B, C, D, E, H, L, SP, PC
- Flags: S(7) Z(6) AC(4) P(2) CY(0) — bit positions
- Interrupt: INTR with RST vector (standard RST 7 for USART)
- T-state counting per instruction
- Separate `IN`/`OUT` I/O space (port 0x19 = USART data, 0x28 = USART status)

### Memory Map
```
$0000-$1FFF  ROM1 (D2764A) — init, HPGL parser, serial I/O
$2000-$3FFF  ROM2 (D2764A) — main logic, math, Bresenham
$4000-$5FFF  ROM3 (D2764A) — plotter routines, font tables
$6000-$67FF  RAM (2KB КР537РУ10)
$E000-$E3FF  PIO1 (КР580ВВ55 / i8255)
$E400-$E7FF  PIO2 (КР580ВВ55 / i8255)
$E800-$EBFF  PIT  (КР580ВИ53 / i8253)
$EC00-$EFFF  SIO  (КР580ВВ51А / i8251)
```

### Peripherals

**PIO1 (КР580ВВ55, i8255)** — Mode 0, Control=$92
```
PA = INPUT  — keyboard rows (PA0-PA5)
PB = INPUT  — DIP switches (PB4-PB7) + limit switches (PB0-PB3)
PC = OUTPUT — keyboard columns (PC0-PC1), LEDs (PC2-PC5)
```

**PIO2 (КР580ВВ55, i8255)** — Mode 0, Control=$80
```
PA = OUTPUT — stepper motor X phases
PB = OUTPUT — ExecBoard control signals
PC = OUTPUT — pen control (PC0-PC3), timer CE (PC7)
```

**PIT (КР580ВИ53, i8253)**
```
Counter 0: mode 3 (square wave) → USART baud clock
Counter 1: mode 3 (square wave) → Buzzer
```

**USART (КР580ВВ51А, i8251)**
```
RxD, TxD, RTS, CTS, DTR, DSR — full RS-232 handshake
XOn-XOff flow control
```

### Address Decoder (U3 К555ИД7 = 74LS154)
```
Y0  $E000-$E3FF  PIO1
Y1  $E400-$E7FF  PIO2
Y2  $E800-$EBFF  PIT
Y3  $EC00-$EFFF  SIO
```

### RAM Variables (firmware writes debug duplicates here)
```
$6180-$6181  X_POS        current X (LO, HI)
$6186-$6187  Y_POS        current Y (LO, HI)
$6188-$618B  X/Y_TARGET   destination coordinates
$61C8-$61CB  X/Y_DELTA    Bresenham deltas
$61E8        PEN_COLOR    pen number 0-6
$63F0        PEN_STATE    bit 0: pen down
$63F2        LED_STATE    mirrors PIO1.PC LED bits
```

### Plotter
- XY stepper motors, 8-phase half-step ring: `[1,3,2,6,4,C,8,9]`
- 7 pens, colors from firmware analysis
- Working area: X=0..17200, Y=0..12200 (HPGL units)
- Bresenham line interpolation between points
- Limit switches: Xmin, Xmax, Ymin, Ymax, PenUp, PenDn

### Keyboard Matrix (6 rows × 2 columns = 12 keys)
- Rows PA0-PA5, columns PC0-PC1
- Firmware scans: drive column low, read rows → detect key

### LEDs (4)
- PIO1.PC2-PC5, active high (1=ON)

### DIP Switches (4)
- PIO1.PB4-PB7, configure baud rate & protocol

## Key Design Decisions for Rust

1. **CPU core**: trait-based with pluggable memory/I/O callbacks
2. **Opcode table**: generated at compile-time via const fn or macro
3. **MMU**: generic over backend (Vec-backed for tests, wasm-backed for web)
4. **Disassembler**: can share opcode table metadata with CPU
5. **I/O devices**: trait objects behind an IoBus
6. **Plotter**: event-driven state machine (line segments → canvas)
7. **HPGL parser**: nom or handwritten parser

## Porting Strategy

| Phase | Language | Deliverable |
|-------|----------|-------------|
| 1 | Rust (native) | CPU+MMU+disasm working, CLI debugger |
| 2 | Rust (wasm) | Web UI via wasm-bindgen + canvas |
| 3 | Python | PyO3 bindings to Rust core, Python scripting API |
| 4 | Go | CGo bindings or full rewrite for server-side headless mode |
