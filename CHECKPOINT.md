### Browser Debug Simulator (`sim/`)

| Файл | Описание |
|------|----------|
| `cpu8080.js` | Полный эмулятор К580ИК80, табличный (256 опкодов), флаги, стек, прерывания |
| `memory.js` | MMU: $0000-$5FFF ROM, $6000-$63FF RAM, $E000+ I/O (PPI, PIT, USART) |
| `main.js` | Склейка: I/O-устройства, дизассемблер, UI, плоттер, загрузка ROM |
| `index.html` | Интерфейс: регистры, дизассемблер, дамп памяти, холст плоттера |
| `styles.css` | Тёмная тема |

#### Возможности отладчика
- **Step / Run / Pause / Reset** — пошаговое выполнение и непрерывный режим
- **Регулятор скорости** — от 100 Hz до max (без задержки)
- **Регистры CPU** — A,B,C,D,E,H,L,SP,PC,флаги (S,Z,AC,P,CY)
- **Дизассемблер** — следование за PC, поиск адреса, брейкпоинты (клик по строке)
- **Дамп памяти** — просмотр по любому адресу
- **Панель I/O** — состояние PPI1/PPI2, PIT (таймер), USART
- **Плоттер** — визуализация шаговых двигателей, перо вверх/вниз, номера пера
- **Автозагрузка ROM** — при запуске через HTTP сервер

### Карта памяти

```
$0000-$1FFF  Chip 1 — ресет, инициализация, низкоуровневые подпрограммы
$2000-$3FFF  Chip 2 — основная логика
$4000-$5FFF  Chip 3 — подпрограммы плоттера + таблицы шрифтов
$6000-$63FF  RAM (стек SP=$6140, рабочие переменные)
$E000-$E3FF  PPI1 (КР580ВВ55А #1) — X-шаговик, Y-шаговик
$E400-$E7FF  PPI2 (КР580ВВ55А #2) — перо, вспомогательное
$E800-$EBFF  PIT  (КР580ВИ53) — 3 канала таймера
$EC00-$EFFF  USART (КР580ВВ51А)
```

### Статистика дизассемблирования

- Инструкций: 9693
- Меток: 736 (формат: `F_L_XXXX` / `F_M_XXXX` / `F_P_XXXX`)
- Областей данных (шрифты): 512 байт
- Инструкций RET: 209
- Инструкций CALL/JMP/переходов: ~1680
- OUT/IN: 3 (основной I/O через memory-mapped STA/LDA)

### Как продолжить

**Запуск симулятора:**
```bash
cd ~/work/Antigravity/github/aftograf && python3 -m http.server 8080
# open http://localhost:8080/sim/
```

**Типичные следующие шаги:**
1. Переименовать метки в осмысленные имена (по анализу кода)
2. Добавить комментарии к ключевым функциям
3. Разобрать структуру данных (таблицы шрифтов в $5E00-$5FFF)
4. Сопоставить порты I/O с реальной периферией
5. Запустить симулятор и пройти инициализацию CPU
6. Восстановить исходный .asm для ресборки (crasm/asm8080)

### Формат меток

- `RESET` — вектор ресета ($0000)
- `F_L_XXXX` — Chip 1 ($0000-$1FFF)
- `F_M_XXXX` — Chip 2 ($2000-$3FFF)
- `F_P_XXXX` — Chip 3 ($4000-$5FFF)
