# Відладчик-симулятор Автограф-882 v1.0.18

![Автограф-882 — Оригінальний пристрій](images/%D0%90%D0%B2%D1%82%D0%BE%D0%B3%D1%80%D0%B0%D1%84_882.01-1990.jpg)
*Оригінальний графопобудовник Автограф-882*

## Можливості

### Емуляція CPU (Rust & Go)
- Повна емуляція К580ІК80А / Intel 8080 — всі 256 опкодів
- Регістри: A, B, C, D, E, H, L, SP, PC (редагуються hex-поля)
- Прапорці: S, Z, AC, P, CY (клікабельні)
- Обробка переривань (INTR з вектором RST 7)
- Лічильник тактів
- Множник швидкості: 1x/10x/100x/1Kx/10Kx/100Kx (Go)

### Пам'ять
- ПЗП: 24 КБ за адресою `$0000–$5FFF` (D2764A)
- ОЗП: 2 КБ за адресою `$6000–$67FF` (КР537РУ10)
- I/O: PPI1 за `$E000`, PPI2 за `$E400`, PIT за `$E800`, USART за `$EC00`

### Дизасемблер (Rust & Go)
- Табличний дизасемблер на всі 256 опкодів
- Breakpoint (клік для встановлення/зняття)
- **Follow PC** — поточна інструкція по центру
- Пошук за адресою, кнопки ◀▶
- Клік за адресою — стрибок у переглядач пам'яті
- Копіювання видимого діапазону в буфер (Go: кнопка Copy)

### Перегляд пам'яті (Rust & Go)
- 32 рядки × 16 байт = 512 Б видимо (Go); 64 рядки (Rust)
- Навігація: рядок адреси + Go, ◀▶, HL-кнопка
- Клік за BC:/DE:/HL:/SP: — стрибок до адреси
- Кольорове маркування: ПЗП (коричневий), ОЗП (золотий), I/O (фіолетовий) — Go
- Редагування байтів кліком (Enter для підтвердження)
- ASCII колонка справа

### Периферія (Rust & Go)
- **КР580ВВ55А (PPI8255)**: дві мікросхеми (PPI1, PPI2)
- **КР580ВІ53 (PIT8253)**: 3 лічильники
- **КР580ВВ51А (USART8251)**: буфери, відправлення hex, лог

### Плоттер (Rust & Go)
- Крокові двигуни XY з фаз PPI
- 7 кольорів пера
- Полотно A4, автошкала
- Завантаження HPGL, покрокове виконання

### HPGL
- Команди: IN, SP, PU, PD, PA, PR
- Preview: малювання всього файлу
- Кроковий режим: ▶ Далі / ▶▶ Все / ⟲ Скинути
- Прогрес-бар

### USART (Go)
- Поле hex-введення, відправлення байтів у CPU
- Лог прийому (останні 20 записів)
- Індикатори TXRDY/RXRDY

### Go GUI: жива симуляція апаратури та налагодження I/O
- Вкладка `Debug` об'єднує CPU, стек і breakpoint-и
- Вкладка `I/O` показує PPI1/PPI2, PIT, USART і зовнішню апаратуру по колонках
- Вкладка `Hardware` містить клавіатурну матрицю 6×2, чотири кінцеві вимикачі X/Y, чотири DIP-входи та LED PPI1.C2–C5
- Клавіші, кінцеві вимикачі та DIP можна змінювати під час виконання CPU
- `Stop on peripheral access` зупиняє Go після інструкції, що звернулася до PPI, PIT або USART
- Рядок події показує `READ/WRITE`, адресу або прямий порт, значення, пристрій і призначення регістра
- Кнопка `?` у вкладці I/O пояснює карту адрес периферії

### Діагностика
- Панель CPU з лічильником тактів
- Стек (8 слів у Go)
- LED плотера на PPI1.C2–C5 (Go)
- Симуляція клавіатури 6×2, кінцевих вимикачів X/Y і DIP-входів (Go)
- Збереження/завантаження сесії в JSON (Go)
- Гарячі клавіші: Space/→ крок, R скидання, F5 пуск/пауза, B breakpoint, ? допомога

## Збірка та запуск

### Rust (основна версія)

```bash
cd rust
cargo run --release
```

Тести:

```bash
cd rust
cargo test -- --test-threads=1
```

### Go (активно розвивається)

```bash
cd go
./trygo.sh
```

`trygo.sh` збирає GUI, запускає unit-тести та GUI smoke-тест, а потім відкриває симулятор. Для прямого запуску: `go run ./cmd/aftograf`. Версія Go використовує Fyne v2.5 і потребує дисплейний сервер (X11/macOS/Wayland).

Тести:

```bash
cd go
go test -count=1 ./...
go test -race ./pkg/app
go vet ./...
```

### Браузерна версія

`sim/` — стара версія для браузера:

Поточна браузерна збірка: `v0.0.7`.

```bash
cd sim && ./tryjs.sh
# Або вручну:
python3 -m http.server 8080
# Відкрий http://localhost:8080/sim/
```

`tryjs.sh` перебудовує bundle, запускає регресійні тести HPGL для `PU/PD`, `PA` і `PR`, а потім запускає локальний сервер.

## Структура проекту

```
├── rust/                  ← Основна версія (Rust)
│   ├── Cargo.toml
│   └── src/ (cpu, memory, disasm, plotter, hpgl, ppi8255, pit8253, usart8251, settings, session)
├── go/                    ← Go версія (Fyne)
│   ├── go.mod / go.sum
│   ├── cmd/aftograf/main.go
│   └── pkg/ (app, cpu, memory, disasm, plotter, hpgl, ppi8255, pit8253, usart8251, settings)
├── sim/                   ← Браузерна версія
├── docs/                  ← Документація
└── images/                ← Скріншоти
```

## Гарячі клавіші

| Клавіша | Дія |
|---------|-----|
| `Пробіл` / `→` | Крок |
| `R` | Скидання CPU |
| `F5` | Пуск / Пауза |
| `B` | Breakpoint |
| `?` | Допомога |

У вкладці Go `I/O` увімкніть `Stop on peripheral access`, щоб зупинитися після поточної інструкції читання або запису периферії. Опис останнього звернення залишається видимим.
